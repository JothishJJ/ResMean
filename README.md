## What This Project Is

ResuMean is a modern web app for job seekers who want concrete feedback before applying. It analyzes a resume PDF and returns a structured report designed to improve readability, recruiter relevance, and ATS performance.

This repository contains a full-stack Next.js app with:
- A marketing-style landing page
- An interactive resume analysis flow
- A server API route that sends the uploaded PDF to a Gemini model

## Core Features

- PDF resume upload workflow
- AI-generated overall match score (`0-100`)
- Match-level summary and executive feedback
- Critical improvement recommendations
- Key strengths extraction
- ATS optimization scoring + explanation
- Keyword suggestions to improve discoverability
- Responsive UI built with App Router and Tailwind CSS

## Tech Stack

- `Next.js 16` (App Router)
- `React 19`
- `TypeScript`
- `Tailwind CSS 4`
- `Vercel AI SDK` (`ai`)

## Quick Start

### 1. Install dependencies

```bash
pnpm install
```

### 2. Configure environment variables

Create `.env.local` in the project root:

```bash
GOOGLE_GENERATIVE_AI_API_KEY=your_google_ai_api_key
```

### 3. Run locally

```bash
pnpm dev
```

Open `http://localhost:3000`.

## Available Scripts

- `pnpm dev` - start local development server
- `pnpm build` - build for production
- `pnpm start` - run the production build
- `pnpm lint` - run ESLint

## API Behavior

Endpoint: `POST /api/analyze`

Request:
- `multipart/form-data`
- Field: `file` (PDF)

Response:
- JSON object with:
  - `score`
  - `matchLevel`
  - `summary`
  - `improvements[]`
  - `strengths[]`
  - `atsOptimization { score, feedback }`
  - `keywordSuggestions[]`

## Notes

- The analyzer route currently targets `gemini-2.5-flash`.
- Keep API keys server-side only (`.env.local`) and never expose them in client code.
- For best results, upload text-based PDFs rather than scanned image-only files.
